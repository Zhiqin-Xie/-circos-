## 1. ANI计算详解

### 1.1 核心概念

**ANI（Average Nucleotide Identity）是全基因组范围内的加权平均相似度，不是局部比对或固定长度片段的相似性。**

### 1.2 详细计算过程

#### 步骤1：基因组分片（固定长度）

```
基因组A → 切成多个1020bp的片段（默认）

例如：
  3Mb基因组 → 约3000个片段

  片段1: position 0 - 1020bp
  片段2: position 1020 - 2040bp
  片段3: position 2040 - 3060bp
  ...
```

#### 步骤2：片段映射（全局搜索）

```
每个片段 → 在整个基因组B中寻找最佳匹配位置

⚠️ 注意：
  - 不是固定位置比对
  - 而是在整个基因组范围搜索
  - 可以匹配到任何位置

例如：
  A的片段1（位置100k-101k）→ 可能匹配到B的位置500k
  A的片段2（位置200k-201k）→ 可能匹配到B的位置50k

   这考虑了基因组重排的情况
```

#### 步骤3：计算每个片段的相似度

```
对每个成功匹配的片段：

片段i vs 最佳匹配区域 → 序列一致性(%)

例如：
  片段1 → 98.5%（匹配良好）
  片段2 → 97.2%（匹配良好）
  片段3 → 无匹配（相似度<70%，丢弃）#未匹配的片段（相似度太低，丢弃）不参与ANI值的计算
  片段4 → 99.1%（匹配良好）
  ...
```

#### 步骤4：加权平均

```
公式：
  ANI = Σ(每个片段的相似度 × 片段长度) / Σ(成功比对的总长度)

例如：
  总片段数：3000个
  成功匹配：2800个
  未匹配：200个

  ANI = (片段1的98.5% + 片段2的97.2% + ... + 片段2800的99.1%) / 2800
      = 98.5%
```

---

### 1.3 关键特点

#### 特点1：是全局性的

```
ANI考虑的是整个基因组的平均情况
不是某个局部区域的相似度

示例：
  基因组A和B有一个区域完全不同（相似度40%）
  → 这会降低ANI

  但如果其他99%的区域都很相似（99%）
  → ANI仍然很高
  → 最终ANI ≈ 98%
```

#### 特点2：不是固定位置比对

```
片段可以匹配到基因组的任何位置

这意味着ANI能够处理：
  ✓ 基因组重排（rearrangement）
  ✓ 倒位（inversion）
  ✓ 易位（translocation）

例如：
  基因组A：  ═══[基因X]═══[基因Y]═══
  基因组B：  ═══[基因Y]═══[基因X]═══
                  ↑ 顺序颠倒

  fastANI仍能识别：基因X和基因Y都存在且相似
  → ANI会很高（~98%）
```

#### 特点3：有比对率的概念（成功匹配的片段才会影响最终的ANI的计算）

```
不是所有片段都能找到匹配

例如：
  基因组A有3000个片段
  成功匹配：2800个（93.3%比对率）
  未匹配：200个（6.7%）

未匹配片段可能是：
  - A特有的基因
  - 相似度太低（<70%）
  - 插入/缺失区域
  - 水平基因转移获得的片段
```

##### 未匹配片段不直接影响ANI数值，但通过比对率反映了基因组的整体差异！

---

### 1.4 输出解读

#### fastANI标准输出

```
query.fna  ref.fna  98.5  2800  3000

字段解读：
  字段1：query.fna        查询基因组文件名
  字段2：ref.fna          参考基因组文件名
  字段3：98.5            ANI值（%）
  字段4：2800            成功比对的片段数
  字段5：3000            总片段数

计算：
  ANI = 98.5%
  比对率 = 2800/3000 = 93.3%
```

#### ANI值的生物学意义

| ANI范围      | 分类学意义        | 说明                |
| ---------- | ------------ | ----------------- |
| **> 95%**  | 同一种（species） | 公认的种界定标准          |
| **90-95%** | 可能同一种        | 边界区域，需要其他证据       |
| **75-90%** | 同一属（genus）   | 不同种，但同属           |
| **< 75%**  | 计算失败         | fastANI无法计算（差异太大） |

#### 实际案例

```
案例1：同一菌株的不同分离物
  Escherichia coli K12 vs E. coli O157:H7
  ANI = 98.2%
  → 同一种，不同菌株

案例2：同属不同种
  Staphylococcus aureus vs S. epidermidis
  ANI = 76.3%
  → 同一属，不同种

案例3：不同属
  Staphylococcus aureus vs Streptococcus pyogenes
  ANI = 计算失败（NaN）
  → 差异太大，fastANI无法计算
```

## 2. nucmer比对原理

### 2.1 核心算法

**nucmer使用后缀树（suffix tree）+ 最大唯一匹配（MUM, Maximum Unique Match）算法进行全基因组精确比对。**

---

### 2.2 详细比对过程

#### 步骤1：建立后缀树

```
将基因组A的所有子序列构建成索引结构

例如序列"ATCGATCG"的所有后缀：
  ATCGATCG
  TCGATCG
  CGATCG
  GATCG
  ATCG
  TCG
  CG
  G

后缀树将这些后缀组织成树状结构
→ 支持快速查找任意子序列
```

**后缀树的作用**：

```
传统方法：
  查找序列"ATCG"是否存在
  → 需要逐个比较，O(n)时间复杂度

后缀树：
  查找序列"ATCG"是否存在
  → 树的路径查找，O(m)时间复杂度
  → m是查询序列长度，与基因组长度无关
```

#### 步骤2：查找MUM（Maximum Unique Match）

```
对基因组B的每个位置 → 在A的后缀树中查找最长匹配

MUM要求：
  ✓ 匹配长度 ≥ 20bp（默认最小MUM长度）
  ✓ 在基因组A中唯一（避免重复序列）
  ✓ 在基因组B中唯一

例如：
  位置: B[200,000]
  查找: 最长匹配 → 找到A[100,000]
  长度: 50bp
  验证: 在A和B中都是唯一的
  → 这是一个MUM
```

**为什么要求"唯一"**：

```
避免重复序列的干扰

例如：
  基因组A有10个拷贝的rRNA基因（完全相同）
  基因组B的rRNA基因可以匹配A的任何一个拷贝
  → 无法确定真正的同源关系
  → 排除这种"非唯一"的匹配
```

#### 步骤3：扩展MUM成为完整比对

```
MUM只是"种子"，需要向两端扩展

初始MUM：
  A[100,000-100,050] ↔ B[200,000-200,050]
  长度：50bp

向左扩展：
  检查 A[99,999] vs B[199,999]
  如果匹配 → 继续向左
  如果不匹配 → 停止

向右扩展：
  检查 A[100,051] vs B[200,051]
  如果匹配 → 继续向右
  如果不匹配 → 停止

最终区域：
  A[95,000-155,000] ↔ B[195,000-255,000]
  长度：60kb（扩展了1200倍！）
```

**扩展过程示例**：

```
原始MUM（种子）：
  AAAAATCGATCGAAAAA
  ↑              ↑
  起点           终点
  50bp

向两端扩展（允许错配）：
  GGGGGAAAAATCGATCGAAAAACCCCC
  ↑                         ↑
  扩展起点                  扩展终点
  60kb（允许少量错配）
```

#### 步骤4：计算比对质量

```
对每个扩展后的区域，计算：

1. 错配数（Mismatches）
   ATCG
   ATTG  → 1个错配（C→T）

2. 插入缺失数（Indels）
   ATCGATCG
   ATC-ATCG  → 1个缺失（G删除）

3. 总错误数（Errors）
   errors = mismatches + indels

4. 序列一致性（Identity）
   identity = (1 - errors / length) × 100%
```

**实际案例**：

```
区域长度：60,000 bp
错配数：300个（SNPs）
插入缺失：150个
总错误：450

计算：
  identity = (1 - 450/60000) × 100%
           = (1 - 0.0075) × 100%
           = 99.25%
```

### 2.3 输出：Delta文件

#### Delta文件格式

```
示例内容：
/path/to/genome_A.fna /path/to/genome_B.fna
NUCMER
>genome_A_scaffold1 >genome_B_scaffold1 2814816 2750948
95000 155000 195000 255000 450 15 8
300000 350000 450000 500000 800 20 10
800000 850000 100000 150000 600 18 7
0
>genome_A_scaffold2 >genome_B_scaffold2 1500000 1450000
...
```

#### 逐行解读

```
行1-2：文件头
  /path/to/genome_A.fna /path/to/genome_B.fna
  NUCMER

行3：scaffold信息
  >genome_A_scaffold1 >genome_B_scaffold1 2814816 2750948

  含义：
    - A的scaffold1 长度2,814,816 bp
    - B的scaffold1 长度2,750,948 bp

行4：比对区域1
  95000 155000 195000 255000 450 15 8

  含义：
    95000, 155000  → A的起止位置（60kb区域）
    195000, 255000 → B的起止位置（60kb区域）
    450           → 错配数（mismatches + indels）
    15            → 未比对的nucleotide数
    8             → 比对段数（可能有gap分成多段）

  计算identity：
    length = 155000 - 95000 = 60000
    identity = (1 - 450/60000) × 100% = 99.25%

行5：比对区域2
  300000 350000 450000 500000 800 20 10

  含义：
    A[300k-350k] ↔ B[450k-500k]
    50kb区域，800个错误
    identity = (1 - 800/50000) × 100% = 98.4%
```

#### 脚本如何处理Delta文件

```python
def parse_delta(delta_file, min_id=85.0, min_len=500):
    links = []

    for line in delta_file:
        # 解析：r1, r2, q1, q2, errors, ...
        r1, r2, q1, q2 = map(int, parts[:4])
        errors = int(parts[4])

        # 计算长度和相似度
        length = max(abs(r2 - r1), abs(q2 - q1))
        identity = (1 - errors / length) * 100

        # 过滤
        if identity >= min_id and length >= min_len:
            links.append({
                'ref_start': min(r1, r2),
                'ref_end': max(r1, r2),
                'query_start': min(q1, q2),
                'query_end': max(q1, q2),
                'identity': identity,
                'length': length
            })

    return links
```

## 3. 参数详解

### 3.1 MIN_IDENTITY：相似度阈值

#### 参数定义

```python
MIN_IDENTITY = 85.0  # 单位：百分比（%）
```

**含义**：只保留序列一致性 ≥ 85% 的相似区域

#### 计算公式

```
identity = (1 - errors / length) × 100%

其中：
  errors = 错配数 + 插入缺失数
  length = 比对区域长度
```

#### 生物学意义

```
85%相似度表示：
  - 每100bp允许15个差异（SNP或indel）
  - 在DNA水平上是中等严格的标准
  - 适合同种不同菌株的比较

不同阈值的适用场景：

  95%：非常严格
    - 每100bp只允许5个差异
    - 适合：同一菌株或近缘菌株
    - 例如：E. coli K12 vs E. coli MG1655

  85%：中等严格（默认）
    - 每100bp允许15个差异
    - 适合：同一种不同菌株
    - 例如：S. aureus MRSA vs S. aureus MSSA

  75%：宽松
    - 每100bp允许25个差异
    - 适合：同一属不同种
    - 例如：S. aureus vs S. epidermidis
```

#### 实际影响

```
50个基因组的比对结果：

MIN_IDENTITY = 95%：
  - 保留区域：300个
  - 连线数：300条
  - 特点：非常精确，只显示高度相似区域

MIN_IDENTITY = 85%（默认）：
  - 保留区域：850个
  - 连线数：850条
  - 特点：平衡，显示大部分真实相似区域

MIN_IDENTITY = 75%：
  - 保留区域：1500个
  - 连线数：1500条
  - 特点：包含更多相似度较低的区域
  - 风险：可能包含假阳性匹配
```

#### 调整建议

```
提高MIN_IDENTITY（更严格）的情况：
  ✓ 基因组非常相似（ANI > 98%）
  ✓ 想突出高度保守区域
  ✓ 图上连线太多太乱

  推荐值：90-95%

降低MIN_IDENTITY（更宽松）的情况：
  ✓ 基因组差异较大（ANI 75-90%）
  ✓ 想探索更多潜在相似区域
  ✓ 图上连线太少

  推荐值：75-80%

  ⚠️ 注意：< 75%可能产生很多假阳性
```

### 3.2 MIN_LENGTH：最小区域长度

#### 参数定义

```python
MIN_LENGTH = 500  # 单位：碱基对（bp）
```

**含义**：只保留长度 ≥ 500bp 的相似区域

#### 生物学意义

```
500bp的设置表示：
  - 约等于一个中等大小的基因
  - 可以代表真实的功能单元
  - 过滤掉偶然的短片段匹配

不同长度的生物学参考：
  - 100bp：非常短，可能是偶然匹配
  - 500bp：中等基因的长度（默认）
  - 1000bp：完整的小基因
  - 5000bp：多个基因的片段
  - 10kb+：基因簇或操纵子
```

#### 过滤效果

```
某对基因组的nucmer原始结果：
  总区域数：1000个

  按长度分布：
    ≥ 10kb：    50个区域
    1kb-10kb：  150个区域
    500bp-1kb： 200个区域
    100bp-500bp：400个区域
    < 100bp：   200个区域

设置MIN_LENGTH = 500bp：
  保留：50 + 150 + 200 = 400个区域
  丢弃：600个短片段

  结果：
    - Circos图上绘制400条连线
    - 图清晰，重点突出
```

#### 不同阈值的效果

```
MIN_LENGTH = 100bp（宽松）：
  保留：1000个区域
  效果：
    ✓ 包含所有相似片段
    ✗ 图上非常拥挤
    ✗ 包含很多短片段"噪音"
    ✗ 难以识别重要的相似区域

MIN_LENGTH = 500bp（默认）：
  保留：400个区域
  效果：
    ✓ 过滤了大部分短片段噪音
    ✓ 保留了真实的同源区域
    ✓ 图清晰易读

MIN_LENGTH = 5000bp（严格）：
  保留：200个区域
  效果：
    ✓ 只显示长片段相似区域
    ✓ 图非常清晰
    ✗ 可能丢失很多真实的相似基因
    ✗ 低估了实际的相似程度
```

#### 调整建议

```
增大MIN_LENGTH的情况：
  ✓ 图上连线太多太乱
  ✓ 只关注大片段重排或保守区域
  ✓ 基因组很相似（有很多短匹配）

  推荐值：1000-5000bp

减小MIN_LENGTH的情况：
  ✓ 图上连线太少
  ✓ 想看到更多细节
  ✓ 基因组差异较大（只有少量匹配）

  推荐值：200-500bp

  ⚠️ 注意：< 100bp会产生大量噪音
```

### 3.3 两个参数的组合效果

#### 组合策略

```
严格设置（高质量，少连线）：

    - 非常近缘的基因组（ANI > 98%）

标准设置（平衡）：
  MIN_IDENTITY = 85%
  MIN_LENGTH = 500bp

  适用场景：
    - 同种不同株比较（ANI 90-98%）
    - 常规分析
    - 平衡精确性和完整性

  效果：
    ✓ 图清晰可读
    ✓ 显示大部分真实相似区域

宽松设置（探索性，多连线）：
  MIN_IDENTITY = 75%

  适用场景：
    - 远缘基因组（ANI < 90%）
    - 探索性分析

  效果：
    ✓ 包含更多潜在相似区域
    ✗ 图可能拥挤
    ✗ 可能包含假阳性
```

## 4. 总结对比

### 4.1 ANI vs nucmer 核心区别

| 维度       | fastANI           | nucmer             |
| -------- | ----------------- | ------------------ |
| **算法**   | k-mer匹配 + 加权平均    | 后缀树 + MUM + 扩展     |
| **计算范围** | 全基因组平均            | 所有局部区域             |
| **输出类型** | 单个数字              | 多个区域列表             |
| **输出示例** | "98.5%相似"         | "100个区域，每个有位置和相似度" |
| **计算时间** | 快（10-30秒/对）       | 较慢（5-15秒/对）        |
| **用途**   | 快速评估总体相似性         | 精确定位具体相似片段         |
| **在脚本中** | 生成CSV矩阵（参考）,不影响画图 | 决定Circos图的连线       |

### 4.2 形象比喻

```
ANI就像：
  两个人的"总体相似度"
  "他们长得90%像"
  → 给出一个整体评价

nucmer就像：
  详细列出相似的部分
  "鼻子长得像（95%），眼睛长得像（92%），嘴巴不像（60%）"
  → 给出每个部位的详细比较
```

### 4.3 两者关系

```
互补关系：

ANI提供全局视角：
  "这两个基因组整体上98.5%相似"
  → 快速判断：同一种

nucmer提供局部细节：
  "有100个相似区域"
  "最大的相似区域在A的100-200kb，与B的500-600kb对应"
  "这个区域99.2%相似，长度100kb"
  → 精确定位：哪里相似

在脚本中的配合：
  1. ANI告诉你整体相似度（CSV文件）
  2. nucmer找出具体相似位置（Circos连线）
  3. 两者结合给出完整图景
```

### 4.4 参数含义总结

| 参数               | 含义    | 单位  | 生物学意义    | 调整效果        |
| ---------------- | ----- | --- | -------- | ----------- |
| **MIN_IDENTITY** | 相似度阈值 | %   | 允许的差异程度  | ↑ 连线减少（更严格） |
| **MIN_LENGTH**   | 最小长度  | bp  | 有意义的片段大小 | ↑ 过滤短片段     |

### 4.5 实际应用建议

```
场景1：近缘菌株比较（ANI > 95%）
  推荐设置：
    MIN_IDENTITY = 90-95%
    MIN_LENGTH = 1000-5000bp

  原因：
    - 基因组很相似，有大量匹配
    - 需要严格过滤突出重要区域
    - 避免图太乱

场景2：同种不同株（ANI 90-95%）
  推荐设置：
    MIN_IDENTITY = 85%
    MIN_LENGTH = 500bp（默认）

  原因：
    - 相似度适中
    - 平衡完整性和清晰度
    - 最常用场景

场景3：远缘比较（ANI < 90%）
  推荐设置：
    MIN_IDENTITY = 75-80%
    MIN_LENGTH = 200-500bp

  原因：
    - 相似区域较少
    - 需要宽松设置找到相似区域
    - 可能需要多次调整

场景4：探索性分析
  推荐设置：
    先宽松：MIN_IDENTITY = 75%, MIN_LENGTH = 100bp
    再严格：MIN_IDENTITY = 90%, MIN_LENGTH = 5000bp
    对比两种结果

  原因：
    - 了解相似度范围
    - 确定最佳参数
```

## 5. 常见问题

### Q1：ANI和nucmer的identity有什么区别？

```
ANI：
  - 全基因组的加权平均值
  - 一个数字（如98.5%）
  - 考虑所有成功匹配的片段

nucmer identity：
  - 每个局部区域的相似度
  - 每个区域一个值
  - 可能有100个不同的identity值

示例：
  ANI = 98.5%（整体）
  nucmer找到100个区域：
    - 区域1：99.2%
    - 区域2：98.8%
    - 区域3：96.5%
    - ...
    - 平均后接近ANI值
```

### Q2：MIN_IDENTITY设置多少合适？

```
参考标准：

95%+：非常严格
  - 适合：同一菌株或极近缘
  - 可能丢失：一些真实的相似区域

85%（默认）：标准设置
  - 适合：大多数情况
  - 平衡：精确性和完整性

75%：宽松设置
  - 适合：远缘比较
  - 风险：可能包含假阳性

建议：
  1. 先用默认85%运行
  2. 如果连线太多 → 提高到90-95%
  3. 如果连线太少 → 降低到75-80%
```

### Q3：MIN_LENGTH应该设多大？

```
参考标准：

5kb+：非常严格
  - 只保留大片段
  - 图非常清晰
  - 可能低估相似度

500bp（默认）：标准设置
  - 过滤短片段噪音
  - 保留真实同源区域
  - 适合大多数情况

100bp：宽松设置
  - 包含更多细节
  - 图可能拥挤
  - 适合探索性分析

建议：
  - 基因组相似（很多匹配）→ 用1-5kb
  - 基因组差异大（少量匹配）→ 用500bp或更小
```

### Q4：为什么ANI计算失败但nucmer有结果？

```
原因：
  ANI（fastANI）有相似度下限：
    - ANI < 75% → 计算失败（返回NaN）
    - 差异太大，算法不适用

  nucmer没有严格下限：
    - 只要找到相似区域就输出
    - 即使整体相似度很低

示例：
  不同属的基因组：
    fastANI → 计算失败（NaN）
    nucmer → 找到20个相似区域

  这些区域可能是：
    - 保守的管家基因
    - 水平转移的基因
    - 高度保守的功能基因
```
